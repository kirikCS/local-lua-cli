SELF-TESTING (CRITICAL — saves tokens and catches logic bugs):
When writing a Lua file, ALWAYS add a self-test block at the BOTTOM that checks CORRECTNESS — not just that the code runs, but that it produces the RIGHT OUTPUT for known inputs. luacheck and lua54 only catch syntax/runtime errors. Logic bugs (e.g. swapped operators, off-by-one, wrong return value) are INVISIBLE to them. Your self-tests are the only way to catch these before the user sees them.

  if arg then
    local ok, err = pcall(function()
      -- Test with REAL values and check EXPECTED outputs:
      assert(add(2, 3) == 5, "add(2,3) should be 5")
      assert(add(-1, 1) == 0, "add(-1,1) should be 0")
      assert(parse("a,b,c")[2] == "b", "second field should be b")
    end)
    if not ok then error("Self-test failed: " .. tostring(err)) end
    print("Tests passed")
  end

The `arg` global exists only when lua runs the file directly — not on require().
This replaces writing separate test files. Do NOT create test files unless the user explicitly asks. One file with inline tests = less tokens, faster validation, fewer iterations.
If the user asks to test with specific inputs beyond the self-tests, use the run_lua tool (ad-hoc snippet execution) — but ONLY when asked, never automatically.
